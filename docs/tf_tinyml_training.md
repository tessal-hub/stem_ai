# TENSORFLOW TO TINYML (ESP32) TRAINING WORKFLOW

## 1. Core Principles for Microcontrollers
- **Memory Limits:** ESP32 has limited SRAM (typically 520KB total, much less for the app). Models must be extremely small (target < 50KB - 100KB).
- **Architecture:** Prefer small Dense (Fully Connected) networks, 1D CNNs (for time-series sensor data), or small 2D CNNs (for vision). Avoid heavy RNNs/LSTMs unless strictly necessary.
- **Pipeline:** Data Prep -> Train TF Model -> Convert to TFLite -> Post-Training Quantization (Int8) -> Convert to C-Byte Array -> Deploy.

3. Model Quantization (CRITICAL FOR ESP32)
Microcontrollers lack powerful Floating-Point Units (FPUs) or have limited float precision. You MUST quantize the model from float32 to int8.
```python
Python
# 2. Convert and Quantize the Model
def representative_data_gen():
    # Provide a small representative dataset for quantization
    for i in range(100):
        yield [X_train[i:i+1].astype(np.float32)]

converter = tf.lite.TFLiteConverter.from_keras_model(model)
converter.optimizations = [tf.lite.Optimize.DEFAULT]
converter.representative_dataset = representative_data_gen
# Ensure strictly INT8 operations
converter.target_spec.supported_ops = [tf.lite.OpsSet.TFLITE_BUILTINS_INT8]
converter.inference_input_type = tf.int8
converter.inference_output_type = tf.int8

tflite_model = converter.convert()

# Save the .tflite file
with open("model_quantized.tflite", "wb") as f:
    f.write(tflite_model)
4. Converting to C Array
ESP32 cannot read a .tflite file from a normal PC filesystem. It must be embedded as a C array in the firmware.
```python
Python
# 3. Python script to convert .tflite to C Header file
import os

def convert_tflite_to_c(tflite_path, c_file_path, array_name="g_model"):
    with open(tflite_path, "rb") as f:
        tflite_content = f.read()
    
    hex_lines = [', '.join([f'0x{byte:02x}' for byte in tflite_content[i:i+12]]) for i in range(0, len(tflite_content), 12)]
    hex_array = ',\n  '.join(hex_lines)
    
    with open(c_file_path, "w") as f:
        f.write(f"// Auto-generated by Python\n")
        f.write(f"#include <stdint.h>\n\n")
        f.write(f"const unsigned int {array_name}_len = {len(tflite_content)};\n")
        f.write(f"const unsigned char {array_name}[] __attribute__((aligned(4))) = {{\n  {hex_array}\n}};\n")

# Usage: convert_tflite_to_c("model_quantized.tflite", "model_data.h")
5. Deployment Rules (ESP-IDF / Arduino IDE)
Use the TensorFlowLite_ESP32 library.

Operations resolver: Only load the specific mathematical operations your model uses (e.g., MicroMutableOpResolver<5> resolver; resolver.AddConv2D();) instead of AllOpsResolver to save memory.

Tensor Arena: Allocate an array (constexpr int kTensorArenaSize = 4 * 1024;) for input, output, and intermediate arrays. Tune this size based on the model.

# 3. HARDWARE DATA PACKET (ESP32 MPU6050)

## A. Protocol & Format
- **Communication:** Serial (UART)
- **Sample Rate:** 50Hz (1 sample every 20ms)
- **Format:** CSV String terminated by `\n` (Newline).
- **Structure:** `aX,aY,aZ,gX,gY,gZ`
- **Data Type:** Raw `int16_t` values (String representation). 

## B. Python Parsing Rules (in `/logic`)
1. **Reading:** `serial_worker.py` reads `readline().decode('utf-8').strip()`.
2. **Validation:** Ensure the string splits into exactly 6 parts (`len(parts) == 6`). Drop the packet if corrupted to prevent UI crashes.
3. **Data Conversion & Normalization:**
   - Raw values MUST be converted to `float` in the logic layer.
   - Assuming MPU6050 default ranges (±2g for Accel, ±250°/s for Gyro):
     - `accel_g = raw_accel / 16384.0`
     - `gyro_dps = raw_gyro / 131.0`
4. **Buffering:** Push the normalized data `[ax, ay, az, gx, gy, gz]` to a deque buffer in `data_store.py`. Do NOT send raw strings to the UI.
